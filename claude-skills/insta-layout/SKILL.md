# Instagram レイアウト設計スキル

ベース画像（文字なし／固定要素のみ）とサンプル画像（テキスト追加済み）を見比べて、「どこに・どんなテキストを・どう載せたか」をJSON仕様にまとめる。将来 `insta_image.py` と組み合わせて同じデザインの画像を量産するための土台。

## あなたのロール

ユーザーから2枚の画像パスを受け取り、目視で比較して配置情報を推定し、`~/.claude/layouts/<テンプレート名>.json` に保存する。

## 手順

### 1. 入力を受け取る

ユーザーから以下を確認（引数で渡されていればそれを使う）:

- **ベース画像パス**（文字なし／固定要素のみ）
- **サンプル画像パス**（複数渡されたら複数から共通パターンを抽出）
- **テンプレート名**（省略時はユーザーに候補を提示して決定。ファイル名から推測した候補を3つほど出す）

### 2. 両画像を Read ツールで読み込む

Read ツールに画像パスを渡すと画像が視覚的に表示される。ベース画像とサンプル画像を両方読み込む。

### 3. 差分を目視で特定する

サンプル側にのみ存在する（またはベース側のplaceholderから変化した）テキストゾーンを列挙する。各ゾーンについて以下を推定する:

- **id**: 意味的な名前（例: `vol_label`, `main_copy`, `footer`）
- **位置**:
  - 中央揃えなら `x_center`（画像の水平中心からのxピクセル、通常は画像幅/2）と `y_top`（上端からのyピクセル）
  - 左右端なら `x`（左端からのxピクセル）
  - 画像サイズを基準に相対的に推定
- **font**: `hiragino-w3`〜`hiragino-w9`（太さ） / `hiragino-maru`（丸ゴ） / `hiragino-mincho`（明朝） / `helvetica` / `avenir`
  - 太いゴシック=w8〜w9、中太=w6、細=w3〜w5
- **size**: フォントサイズ（px）。画像高さに対する相対サイズから逆算（例: 画像600pxで文字が60pxなら10%）
- **color**: HEXカラー。黒系なら濃度を判断し `#2a2a2a` など中間値も使う
- **align**: `center` / `left` / `right`
- **text_template**: 可変部分を `{placeholder}` で置換（例: `"Vol. {vol} {subtitle}"`）
- **sample_text**: サンプル画像に実際に書かれていた文字列（改行は `\n`）
- **max_lines**（任意）: 改行の上限行数

### 4. JSON を組み立てて保存する

保存先: `~/.claude/layouts/<テンプレート名>.json`

```json
{
  "template_id": "テンプレート名",
  "base_image": "/abs/path/to/base.png",
  "sample_images": ["/abs/path/to/sample1.png"],
  "canvas": {"width": 480, "height": 600},
  "zones": [
    {
      "id": "vol_label",
      "x_center": 240,
      "y_top": 170,
      "font": "hiragino-w6",
      "size": 26,
      "color": "#4a4a4a",
      "align": "center",
      "text_template": "Vol. {vol} {subtitle}",
      "sample_text": "Vol. 01 パート収入の壁"
    }
  ],
  "notes": "推定根拠・注意点・固定要素（既に焼き込まれている要素）などをメモ"
}
```

### 5. ユーザーに結果を報告する

- 保存パスを伝える
- 検出したゾーン数と各ゾーンの要約（id・位置・テキスト）を箇条書きで提示
- 「位置・フォント・色で調整したい点はありますか？」と確認し、修正依頼があればJSONを編集する

## ゾーン推定のコツ

- **フォントの太さ判定**: 線の太さ／高さ比 で判定。線幅が高さの1/8以上なら w9、1/10前後なら w6、1/14前後なら w3〜w4
- **色の判定**: 真っ黒ではなく `#2a2a2a`〜`#4a4a4a` の濃いグレーのことが多い。白系も `#fefefe` より `#f2f2f2` など微妙に落とすとなじむ
- **位置の目安**: 画像高さを基準に「上から何%」で把握 → ピクセルに変換
- **改行位置**: サンプルの改行をそのまま `\n` で再現。将来の可変テキストでも同様の改行ルールが適用できるようメモに残す

## 複数サンプルから共通パターンを抽出する場合

- 全サンプルで共通の位置・フォント・色 → 固定値として採用
- サンプルごとに異なる部分 → `text_template` で `{placeholder}` 化
- 位置がサンプル間で微妙にズレる → 平均値を採用し notes に幅を記録

## 画像サイズが不明な場合

Bash で確認:
```bash
python3 -c "from PIL import Image; im=Image.open('パス'); print(im.size)"
```

## 出力後の活用（将来）

このJSONは将来、`insta_image.py` の複数テキスト対応版や、新規の `insta-post-auto` スキルが読み込んで、同じレイアウトで文面だけ差し替えた画像を量産するために使う。今回は設計データの生成までがスコープ。
